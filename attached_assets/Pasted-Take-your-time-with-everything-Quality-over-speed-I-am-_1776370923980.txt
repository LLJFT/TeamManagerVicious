When navigating into a roster, the Schedule page sometimes loads but appears frozen (no data, no movement). The user has to leave the tab, go to another tab (like Events), then come back to Schedule for it to work.
Root cause is likely a timing/race condition: the Schedule query fires before rosterId is confirmed, so it returns nothing and hangs.
Fix: Add a guard so the Schedule query does NOT fire until both gameId AND rosterId are confirmed and set. Add a loading skeleton so the UI feels responsive while data loads.
