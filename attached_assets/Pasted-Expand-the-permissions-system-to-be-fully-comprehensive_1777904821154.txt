There are two separate permission scopes in the platform:

1. ROSTER PERMISSIONS — control what a role can do INSIDE a specific roster
   (visible in the roster role editor modal — image 2)
   Currently has: Schedule, Events, Results, Players, Statistics, Chat, Staff, Dashboard, Home
   MISSING: many granular permissions for every roster feature

2. HOME PERMISSIONS — control what a role can access at the platform HOME level
   (visible in User Permissions tab — image 1)
   Currently has: basic view/manage for Dashboard, Calendar, Users, Roles, Game Access, Chat, Settings
   MISSING: many granular permissions

Both must be expanded to cover EVERY feature and action available in the platform.
The goal: Super Admin can control every single permission per role with full granularity.

PART 3 — ENFORCEMENT
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Every new permission must be ENFORCED — not just stored.

For each new permission key:
A) Add it to the permissions schema/type definitions
B) Add it to the default role seeds (Super Admin gets all = true, others get sensible defaults)
C) Enforce it in the relevant API route(s) with a permission check
D) Enforce it in the UI by hiding/disabling the relevant button, tab, or section
   when the user lacks the permission

Do NOT just add checkboxes in the UI without wiring them to actual checks.

IMPORTANT:
- Do NOT break any existing permissions that are already working
- Do NOT reset existing role configurations
- New permissions should default to FALSE for existing roles 
  (except Super Admin which always gets all = true)
- Add the new permissions as additional fields — do not replace existing ones

The permission editor UI must display permissions grouped by section.
Each group has:
- A section header (e.g. "GAMES & SCOREBOARD")
- Checkboxes for each permission in that group
- A "Select all" toggle per group to enable/disable all permissions in that group at once

Currently the UI shows all permissions as a flat grid.
Reorganize into the grouped sections defined above.
Keep the existing visual style (checkboxes, dark theme) — just add grouping and "Select all".
