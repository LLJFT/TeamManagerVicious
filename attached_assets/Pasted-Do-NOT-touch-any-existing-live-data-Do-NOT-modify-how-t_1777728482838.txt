CHANGE 3 — Apply template code behavior: clear roster first

Currently when a roster applies a Game Template code, it loads the 
template data on top of existing roster data.

Change this behavior:

When a user pastes a template code into a roster and confirms apply:
  1. Delete ALL existing roster data first:
     - Players
     - Game Modes
     - Maps
     - Heroes config
     - Opponents
     - Seasons
     - Stat fields
     - Any other config scoped to that roster
  2. Then load all template data into the roster:
     - Players (from template Roster tab)
     - Game Modes
     - Maps
     - Heroes
     - Opponents
     - Seasons
     - Stat fields
     - All other template config
  3. After the apply is complete, the roster runs independently —
     future edits to the template do NOT automatically update this 
     roster. The same template code can be applied again at any time 
     to reset the roster back to the template's current state.
  4. The template code itself does NOT change after being applied —
     it stays exactly the same and can be reused on any roster 
     any number of times.

Show a warning dialog before applying:
  "Applying this template will delete ALL existing data in this 
   roster and replace it with the template data. This cannot be 
   undone. Continue?"
  Buttons: "Yes, apply template" (destructive red) | "Cancel"

If the user cancels, nothing is deleted or changed.
If any uploads fail during apply, still complete the apply but log 
the failures — do not abort the entire operation.
