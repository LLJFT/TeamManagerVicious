IMPROVEMENT 2 — Event Result auto-compute with 5-minute settle timer
═══════════════════════════════════════════

CURRENT BEHAVIOR:
The Event Result field shows "Pending — Auto-computed from the games below. Stays Pending until every game has a recorded result."
But it never actually auto-sets even when games have results, and there is no way to manually set it either.

REQUIRED BEHAVIOR:

1) Auto-compute logic (same as before but clarified):
   - Count wins, losses, draws from all games in the event
   - If wins > losses → Event Result = Win
   - If losses > wins → Event Result = Loss
   - If wins === losses → Event Result = Draw
   - If no games have results → stay Pending

2) NEW: 5-minute settle timer
   - Every time a game result is added or changed inside an event, start (or restart) a 5-minute timer for that event
   - After 5 minutes of no changes to game results in that event, auto-compute and set the event result
   - If another game result is added before the 5 minutes expire, reset the timer
   - This prevents the event result from flipping back and forth while games are still being entered

3) Manual override:
   - The Event Result dropdown must always be editable (not locked/disabled)
   - If the user manually sets the event result, respect it — do NOT auto-overwrite a manual selection
   - Only auto-compute if the current value is still "Pending" OR if it was last set by the auto-compute (not manually)
   - Add a small indicator: "Auto-computed" vs "Manually set" next to the result

4) Implementation approach:
   - Use a server-side timer/debounce (not client-side) so it persists across browser refreshes
   - OR use a database column like event_result_locked_at and a background job / cron that runs every minute
   - The simplest reliable approach: store last_game_change_at on the event, and compute result when last_game_change_at is more than 5 minutes ago and result is still auto-pending
   - A cron job or setInterval on server startup (every 60 seconds) scans events where result = 'pending_auto' and last_game_change_at < now - 5 minutes, then computes and saves

5) Edge cases:
   - Event with 0 games → stay Pending
   - Event with games but all results are empty → stay Pending
   - Event result already manually set → do not overwrite
   - Games added after auto-compute fired → re-evaluate after next 5-minute window

6) UI changes:
   - Show "Will auto-set in ~X min" countdown hint when timer is active
   - After auto-set, show "Auto-computed from games" as a subtitle under the result
   - Allow manual edit at any time without losing the dropdown
