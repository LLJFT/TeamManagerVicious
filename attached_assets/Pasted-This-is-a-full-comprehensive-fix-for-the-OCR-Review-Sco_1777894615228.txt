PART 1 — Replace OCR engine with GPT-4o Vision
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

The current text OCR engine produces garbage output for game screenshots.
Replace it entirely with GPT-4o Vision.

1. Check if OPENAI_API_KEY exists in the environment. Use it if yes. 
   Add it to required env vars and document in replit.md if no.

2. In the OCR upload handler: convert the uploaded image to base64,
   then POST to https://api.openai.com/v1/chat/completions
   with model: "gpt-4o" and detail: "high"

3. Create a file server/ocr-prompts.ts with game-specific prompt templates.
   The game name is already known from the match context (roster → game type).
   Use the correct template based on the game. Add these templates:

   marvel-rivals → stats: kills, deaths, assists, finalHits, damage, damageBlocked, healing, accuracy
   valorant       → stats: kills, deaths, assists, headshots, damage, ADR, KAST, rating
   overwatch      → stats: eliminations, deaths, assists, damageDone, healingDone, objectiveTime
   cs2            → stats: kills, deaths, assists, ADR, HSPercent, damage
   default        → ask GPT-4o to detect stat columns automatically

4. Use this GPT-4o system prompt (substitute {GAME_NAME} and {STAT_COLUMNS} from template):

"You are a {GAME_NAME} scoreboard reader.
Extract the scoreboard data from this image and return ONLY valid JSON with no explanation or markdown.

{
  \"map\": \"map name if visible, null if not\",
  \"mode\": \"game mode if visible, null if not\",
  \"ourScore\": number or null,
  \"opponentScore\": number or null,
  \"ourTeam\": [
    {
      \"name\": \"player name exactly as shown\",
      \"hero\": null,
      {STAT_COLUMNS}
    }
  ],
  \"opponentTeam\": [same structure as ourTeam]
}

Rules:
- Our team = top or left section of scoreboard. Opponent = bottom or right section.
- Copy player names exactly as shown including numbers and special characters.
- hero is ALWAYS null — do not attempt to detect heroes from icons.
- For scores: look for small round/point win numbers (1-10 range), NOT kill totals.
- Use null for any value you cannot clearly read. Never guess.
- Return valid JSON only, nothing else."

5. Map GPT-4o JSON response to the existing parsed_candidate structure:
   - ourTeam → side: "us"
   - opponentTeam → side: "opponent"
   - map name → fuzzy match to known maps in DB
   - Store the raw GPT-4o JSON as the raw_ocr field

6. Fallback: if GPT-4o fails for any reason (API error, invalid JSON, empty teams)
   → fall back to old text OCR with toast warning: 
   "Vision AI unavailable — falling back to text OCR"

PROBLEM: Player dropdowns show all users in the entire system, not scoped.

FIX:
A) Our team player dropdown (side = "us"):
   - Show ONLY players from the current roster
   - Never show opponent players or unrelated users

B) Opponent player dropdown (side = "opponent"):
   - Show ONLY players from the opponent's roster linked to this match/event
   - Source: Dashboard → Opponents → View Roster for the opponent in this game
   - If no opponent is linked → show all configured opponents' players for this roster

C) Fuzzy name auto-matching:
   - After OCR returns player names, run fuzzy matching against the scoped player list
   - Use Levenshtein distance (implement inline, no new dependencies needed)
   - Auto-select if similarity > 70% → show badge "Auto-matched (X%)"
   - Pre-select but highlight yellow if similarity 50–70% → show "Low confidence — please verify"
   - Leave empty if similarity < 50% → user picks manually
   - The dropdown is always editable regardless of auto-match confidence

D) No cross-contamination: our team dropdown never shows opponent players and vice versa.
