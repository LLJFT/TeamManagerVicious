In the top navigation bar (the bar visible at the top of the page across all pages), 
add a subscription pill/badge that shows:

1. The subscription TYPE:
   - "TRIAL" (if type is trial)
   - "PAID" (if type is paid/pro)
   Displayed as a colored badge — e.g., yellow/amber for TRIAL, green for PAID

2. A live countdown of time remaining until the subscription ends (endDate):
   - Format: "Xd Xh Xm" (e.g., "30d 4h 22m")
   - The countdown must tick live — update every minute (or every second for the 
     minutes/seconds portion)
   - When less than 24 hours remain, show hours and minutes only (e.g., "4h 22m")
   - When less than 1 hour remains, show minutes only in red (e.g., "22m left")

The subscription data already exists in the database (subscriptions table).
Each subscription has: userId, type ("trial" | "paid"), status ("active" | "expired"), 
startDate, endDate, notes.

- Fetch the current user's active subscription via a new or existing API endpoint
- Recommended: GET /api/subscriptions/me — returns the active subscription for 
  the currently logged-in user (filter by userId = current user, status = "active")
- If no active subscription → do not show the badge (hide it entirely)
- If subscription is expired → do not show it
